# diesel_dart

A type-safe ORM for Dart, inspired by [diesel-rs](https://github.com/diesel-rs/diesel).

> **Status:** early / experimental. APIs may change and it is not yet published to pub.dev. Inspired by
> diesel-rs — not affiliated with it and not a 1:1 port; currently a focused subset. See the
> [diesel-rs comparison](docs/diesel-rs-comparison.md) and [roadmap](docs/ROADMAP.md).

diesel_dart gives you a compile-time-checked query builder, a migration CLI, and code generation for data
classes — the same migration-first workflow as Diesel, adapted to Dart (`build_runner` instead of Rust
derive macros).

## Features

- **Type-safe query builder** — columns and predicates carry a phantom table type, so mixing tables in one
  `WHERE` is a compile error: `from(Users.table).where(Users.age > 18).orderBy(Users.name.asc()).map(...)`.
- **Joins & relations** — `innerJoin` / `leftJoin` (explicit `on:` or FK-driven `onFk:`), self-joins via
  table aliases, and generated self-mapping relation queries from `@Relation`.
- **Async-first** — one `Connection` interface; SQLite today, async backends (Postgres) fit unchanged.
- **Migrations CLI** (`diesel_dart`) — `migration generate/run/revert/redo/list`, `database reset`, tracked
  in `__diesel_schema_migrations`.
- **Schema generation** — `print-schema` introspects the database into a typed Dart schema (dialect-agnostic).
- **Derives** — `@Queryable` (row → object, with relations), `@Insertable` / `@AsChangeset`
  (object → INSERT / UPDATE), generated by `build_runner`.
- **SQLite backend** on [`package:sqlite3`](https://pub.dev/packages/sqlite3) (file or `:memory:`).

## Packages

| Package | Role |
|---|---|
| [`diesel`](packages/diesel) | Dialect-agnostic core: types, schema, expressions, query/write builders, serializer, `Connection`/`SqlDialect`, annotations. |
| [`diesel_sqlite`](packages/diesel_sqlite) | SQLite backend (`SqliteConnection` + `SqliteDialect`). |
| [`diesel_postgres`](packages/diesel_postgres) | Postgres backend — `PostgresConnection` + `PostgresDialect` (on `package:postgres`). |
| [`diesel_cli`](packages/diesel_cli) | The `diesel_dart` executable: migrations + `print-schema`. |
| [`diesel_codegen`](packages/diesel_codegen) | `build_runner` derives for the annotations. |
| [`example/`](example) | End-to-end demo. |

Dart SDK: `>=3.5.0 <4.0.0`.

## Quickstart

Add the dependencies (the packages are unpublished, so use `path:` or `git:` deps for now):

```yaml
dependencies:
  diesel:
  diesel_sqlite:
dev_dependencies:
  diesel_cli:       # provides the `diesel_dart` CLI
  diesel_codegen:   # build_runner derives
  build_runner: ^2.4.0
```

Configure the CLI with a `diesel.yaml` in your project root (or set `DATABASE_URL`):

```yaml
database_url: app.db
migrations_dir: migrations
```

Then follow the migration-first workflow:

```sh
# 1. Create a migration, then edit up.sql / down.sql.
dart run diesel_cli:diesel_dart migration generate create_users

# 2. Apply pending migrations (creates app.db).
dart run diesel_cli:diesel_dart migration run

# 3. Generate the typed schema from the migrated database.
dart run diesel_cli:diesel_dart print-schema -o lib/schema.dart

# 4. Write @Queryable/@Insertable data classes against the schema, then generate mappers.
dart run build_runner build
```

Query and write with full static typing:

```dart
import 'package:diesel/diesel.dart';
import 'package:diesel_sqlite/diesel_sqlite.dart';

import 'schema.dart';   // generated by print-schema
import 'models.dart';   // your @Queryable/@Insertable classes + generated mappers

Future<void> main() async {
  final db = SqliteConnection.open('app.db');

  // INSERT — toInsert() is generated from @Insertable.
  await db.execute(const User(1, 'Bob', 30, 1).toInsert());

  // SELECT — userMapper.read is generated from @Queryable.
  final adults = await db.fetch(
    from(Users.table)
        .where(Users.age > 18)
        .orderBy(Users.name.asc())
        .map(userMapper.read),
  );
  print(adults);

  await db.close();
}
```

See [`example/`](example) for a complete, runnable tour (joins, relations, transactions, raw SQL).

## Status & roadmap

The diesel-rs alignment roadmap **M1–M5 is complete**: SQLite migration compatibility with the Rust `diesel`
CLI, diesel-style API aliases, SQLite query parity, derive parity, and a **full Postgres backend** — SQLite and
Postgres run the same DSL, schema, and migrations unchanged. Remaining/optional work (MySQL, pooling, advanced
Postgres types, schema-first) is in the [roadmap](docs/ROADMAP.md).
